---
title: Set Up a CI/CD Pipeline
description: Automate CVM deployments with GitHub Actions
---

Automate your Phala Cloud deployments with GitHub Actions. Push to main, and your CVM updates automatically.

## Prerequisites

- A GitHub repository with your application code
- A [Phala Cloud account](https://cloud.phala.network/register)
- A Docker Hub or container registry account

The fastest way to start is forking the [cloud-tee-starter-template](https://github.com/Phala-Network/cloud-tee-starter-template) repository.

## Configure Repository Secrets

Go to your repo's **Settings > Secrets and variables > Actions** and add these secrets:

| Secret | Description |
|--------|-------------|
| `PHALA_CLOUD_API_KEY` | From [Phala Cloud Dashboard](https://cloud.phala.network/dashboard) > Avatar > API Tokens |
| `DOCKER_REGISTRY_USERNAME` | Your Docker Hub username |
| `DOCKER_REGISTRY_PASSWORD` | Docker Hub [access token](https://docs.docker.com/docker-hub/access-tokens/) |
| `DOCKER_IMAGE` | Full image path, e.g., `docker.io/username/my-app` |
| `APP_NAME` | CVM name, e.g., `my-tee-app` |

## Create the Workflow

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to Phala Cloud

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Log in to Docker Registry
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_REGISTRY_USERNAME }}
          password: ${{ secrets.DOCKER_REGISTRY_PASSWORD }}

      - name: Build and Push Docker image
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ secrets.DOCKER_IMAGE }}:${{ github.sha }}

      - name: Deploy to Phala Cloud
        uses: Leechael/phala-deploy-action@v2
        with:
          phala-api-key: ${{ secrets.PHALA_CLOUD_API_KEY }}
          cvm-name: ${{ secrets.APP_NAME }}
          compose-file: './docker-compose.yml'
```

## Action Parameters

The `phala-deploy-action` supports these options:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `phala-api-key` | required | Your Phala Cloud API key |
| `cvm-name` | repo name | Name shown in dashboard |
| `compose-file` | `./docker-compose.yml` | Path to compose file |
| `vcpu` | `2` | Number of vCPUs |
| `memory` | `2048` | Memory in MB |
| `disk-size` | `40` | Disk size in GB |
| `envs` | - | Environment variables (YAML format) |
| `app-id` | - | Existing CVM ID (for updates) |

For updates to an existing CVM, set `app-id` to your CVM's app ID. The action will update rather than create.

## Verify Deployment

After the workflow completes, check your [Phala Cloud Dashboard](https://cloud.phala.network/dashboard) to confirm the CVM is running. The endpoint URL appears in the CVM's Network tab.

## Troubleshooting

**Authentication errors**: Double-check that `PHALA_CLOUD_API_KEY` and Docker credentials are set correctly in repository secrets.

**Build failures**: Verify your Dockerfile builds locally with `docker build .` before pushing.

**Debug locally**: Use [act](https://github.com/nektos/act) to run GitHub Actions locally. Create a `.env` file with your secrets and run `act push`.
