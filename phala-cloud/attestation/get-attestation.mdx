You can get attestation reports two ways: check the dashboard for your CVM's default report, or use the dstack SDK to generate fresh quotes programmatically.

Use the dashboard for quick checks. Use the SDK when you need to bind custom data (like public keys or nonces) to your attestation.

## View Attestation Report on Dashboard

Open the **Attestation** tab in your CVM dashboard. Click **Check Attestation** to verify the default report generated during bootstrap.

## Generate Attestation Programmatically

### Prerequisites: Configure Docker Compose

Your application needs access to the dstack service to generate attestation quotes. Mount the dstack socket into your container:

```yaml
version: '3'
services:
  app:
    image: your-app-image
    ports:
      - 8080:8080
    volumes:
      # Mount dstack socket for TEE operations
      - /var/run/dstack.sock:/var/run/dstack.sock
```

### Using dstack SDK

The dstack SDK connects to `/var/run/dstack.sock` automatically. Install for your language:

- **JavaScript/TypeScript:** [@phala/dstack-sdk](https://www.npmjs.com/package/@phala/dstack-sdk)
- **Python:** [dstack-sdk](https://pypi.org/project/dstack-sdk/)
- **Go:** [github.com/Dstack-TEE/dstack](https://github.com/Dstack-TEE/dstack/tree/master/sdk/go)

You can attach custom data to your attestation quoteâ€”up to 64 bytes directly, or unlimited if you hash it first. Use this to bind public keys, nonces, or challenge responses.

<CodeGroup>
```javascript TypeScript
import { DstackClient } from '@phala/dstack-sdk';

// Client automatically connects to /var/run/dstack.sock
const client = new DstackClient();

// Get TEE instance information
const info = await client.info();
console.log('App ID:', info.app_id);
console.log('Instance ID:', info.instance_id);
console.log('App Name:', info.app_name);
console.log('TCB Info:', info.tcb_info);

// Prepare custom data to bind to the attestation
// This can be any application-specific data (e.g., public key, nonce, challenge)
const applicationData = JSON.stringify({
  version: '1.0.0',
  timestamp: Date.now(),
  user_id: 'alice'
});

// Generate remote attestation quote
// SDK handles hashing if data exceeds 64 bytes
const quote = await client.getQuote(applicationData);
console.log('TDX Quote:', quote.quote);
console.log('Event Log:', quote.event_log);

// Replay and verify Runtime Measurement Registers (RTMRs)
// These values prove what software is running in the TEE
const rtmrs = quote.replayRtmrs();
console.log('RTMR0-3:', rtmrs);
```

```python Python
import json
import time
from dstack_sdk import DstackClient

# Client automatically connects to /var/run/dstack.sock
client = DstackClient()

# Get TEE instance information
info = client.info()
print('App ID:', info.app_id)
print('Instance ID:', info.instance_id)
print('App Name:', info.app_name)
print('TCB Info:', info.tcb_info)

# Prepare custom data to bind to the attestation
# This can be any application-specific data (e.g., public key, nonce, challenge)
application_data = json.dumps({
    "version": "1.0.0",
    "timestamp": time.time(),
    "user_id": "alice"
})

# Generate remote attestation quote
# SDK handles hashing if data exceeds 64 bytes
quote = client.get_quote(application_data.encode())
print('TDX Quote:', quote.quote)
print('Event Log:', quote.event_log)

# Replay and verify Runtime Measurement Registers (RTMRs)
# These values prove what software is running in the TEE
rtmrs = quote.replay_rtmrs()
print('RTMR0-3:', rtmrs)
```
</CodeGroup>

Expose this as an API endpoint so clients can request fresh attestation quotes anytime they need proof of your CVM's security.
